# Agent Instructions

- Before starting any change, give the user a short summary of what you will do.
- If a requirement is unclear or conflicts with the current code, ask before coding.
- Search actively for the root cause. Do not hide failures behind silent fallbacks or guesswork.
- When something breaks, surface the exact step, field, or path that failed in a clear error message.
- Respect existing user changes and do not revert unrelated work.
- Finish completed code changes with a git commit.
